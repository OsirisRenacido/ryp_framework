# RYP Framework Internal Workspace

This folder is the self-contained working surface for the installable `ryp_framework` package.

Structure:

- `01_ENTRADAS/TXT/`: input corpus in raw TXT format.
- `01_ENTRADAS/DESCUBRIMIENTOS/`: discovery payloads to be promoted into Serie Q artifacts.
- `04_SERIES_Q/Q_ENTRIES/`: generated Q bundles, one folder per Q ID (`Q001/`, `Q002/`, ...).
- `11_OPERACION/runtime/`: registry and runtime state generated by the framework CLI.

The CLI defaults (`ryp status`, `ryp init-registry`, `ryp scaffold-q`) resolve here so the framework can evolve as its own future workspace.

Each Q bundle currently contains:

- `Qxxx_READINESS.md`
- `Qxxx_EVIDENCE.json`
- `Qxxx_MANIFEST.json`
- `Qxxx_DECISION.json`

The `ryp run-cycle` command applies automatic stage-gate transitions:

- `DRAFT -> READY_FOR_REVIEW`
- `DRAFT -> UNDER_REVIEW`
- `DRAFT -> NEEDS_CURATION`

These transitions are reflected in both the registry and each `Qxxx_MANIFEST.json` file.

TXT ingestion contract:

- `ryp ingest-txt` converts TXT files into discovery JSON/MD payloads.
- `ryp run-all` executes end-to-end pipeline: TXT -> discoveries -> Q bundles.
